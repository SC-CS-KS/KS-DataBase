# Transaction ACID
## 原子性 Atomicity
```md
事务中的所有操作作为一个整体像原子一样不可分割，要么全部成功,要么全部失败。
原子性并不能够完全保证一致性。
```

* 实现
```md
通过日志，将所有对数据的更新操作都写入日志，如果一个事务中的一部分操作已经成功，
但以后的操作，由于断电/系统崩溃/其它的软硬件错误而无法继续，
则通过回溯日志，将已经执行成功的操作撤销，从而达到“全部操作失败”的目的
```

## 持久性 Durability

